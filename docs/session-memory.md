# 会话记忆与经验沉淀

## 启动

新会话应按稳定顺序恢复上下文：项目架构/领域概览、最近会话结果、当前功能或状态总览、与任务相关的 Design/ADR/learnings。只读取与当前任务相关的深层文档，避免把整个历史当成上下文。

## 结束

完成可提交变更时记录：目标、实际改动、关键证据、决策、失败尝试、未完成验证和下一步。系统边界变化时同步架构/功能文档；重大决策进入 ADR；重复验证的模式进入 learnings。

## 三种记忆

- **会话日志**：一次工作的事实、证据和下一步，服务于短期接续。
- **决策记录**：长期有效的选择、上下文、取舍和后果。
- **经验文档**：多次验证的模式、工具使用方式和常见陷阱。

## 提炼规则

- 同一决定被多个会话反复引用时，提取成 ADR。
- 同一模式多次验证后，提取成 learning。
- 同一失败再次出现时，写入明确的“不要再尝试”，并注明重新评估条件。
- 定期归档已被稳定文档吸收的旧日志，避免项目记忆无限膨胀。

记录的目标是降低未来决策成本，而不是制造完整流水账。
